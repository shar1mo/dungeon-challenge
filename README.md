# dungeon-challenge

Приложение читает `config.json` и файл с событиями (`events`), последовательно обрабатывает действия игроков, выводит обработанные события и в конце выводит итоговый `Final report`.

## Что реализовано

- загрузка и валидация конфига
- парсинг времени в формате `HH:MM:SS`
- парсинг событий
- обработка регистрации игроков
- обработка входа и выхода из подземелья
- обработка перемещений между этажами
- обработка убийства монстров
- обработка `boss floor` и убийства босса
- обработка здоровья, лечения, урона и смерти
- обработка `disqualification`
- обработка `impossible move`
- генерация `Final report`
- unit-тесты

## Запуск

- с явным указанием файлов:
```bash
go run ./cmd/dungeon --config testdata/config.json --events testdata/events
```

- для флагов есть дефолтные значения, можно запустить короче:

## Условия и допущения
- `Floors` считается общим количеством этажей, включая этаж босса
- Количество обычных этажей с монстрами равно `Floors - 1`
- `Boss floor` не содержит монстров
- `Dungeon` считается завершённым успешно только если очищены все обычные этажи и убит босс
- `Impossible move` не завершает испытание игрока
- После завершения испытания дальнейшие события игрока игнорируются
- Здоровье игрока не может быть больше 100
- Итоговый отчёт сортируется по `ID игрока`
- В текстах вывода сохранены формулировки из условия, включая опечатки `imposible и recieved`, чтобы вывод совпадал с ожидаемым форматом, чтобы прошли автотесты по формату, если они будут

## Формат входных данных
### `config.json` содержит настройки подземелья:
```json
{
  "Floors": 2,
  "Monsters": 2,
  "OpenAt": "14:05:00",
  "Duration": 2
}
```

**Поля:**
- `Floors` - общее количество этажей, включая boss floor
- `Monsters` - количество монстров на каждом обычном этаже
- `OpenAt` - время открытия подземелья
- `Duration` - длительность работы подземелья в часах


### `events` содержит события в формате:
```text
[HH:MM:SS] playerID eventID extraParam
```

Например:
```text
[14:00:00] 1 1
[14:40:00] 1 2
[14:41:00] 1 3
[14:49:02] 1 11 50
```
`extraParam` может отсутствовать, быть числом или строкой из нескольких слов.

## Формат результата
**Сначала выводятся обработанные события, а затем итоговый отчёт:**
```bash
Final report:
[SUCCESS] 1 [00:24:00, 00:05:00, 00:11:00] HP:35
[FAIL] 2 [00:19:00, 00:00:00, 00:00:00] HP:0
[DISQUAL] 3 [00:00:00, 00:00:00, 00:00:00] HP:100
```

**В отчёте выводятся:**

- состояние игрока: `SUCCESS, FAIL или DISQUAL`
- `ID игрока`
- общее время в подземелье
- среднее время зачистки обычного этажа
- время убийства босса
- здоровье игрока в конце испытания
